# Resonance · Product Principles
# 共振的設計準則

從兩個定位維度出發的功能哲學 —— 為什麼我們這樣做，以及為什麼我們不做那些。

---

## 序：兩個定位維度

所有的功能爭論，最後都會回到兩個定位問題：**使用者的身份要多真實？內容要限定在什麼範圍？**我們的答案是——身份保留彈性，內容徹底鎖定。

### 【定位維度示意】

*   **維度一：使用者身份**
    ```
    完全匿名 ────────────────── [ 帳號真實、展示自選 (保留彈性) ] ────────────────── 完全真實
    ```
*   **維度二：內容類型**
    ```
    虛構・情緒抒發 ─────────────────────────────────────────── [ 有體悟的真實故事 (徹底鎖定) ]
    ```

> *設計哲學：身份是一段可移動的範圍；內容是一個焊死的點。*

值得所有人知道的一件事：*第二個維度已經不是口號，而是架構事實*。每張卡片發布時，系統會用 AI 萃取一份「體悟簽名」——這篇故事的核心領悟、它從什麼處境長出來、帶著什麼樣的情緒色彩——並給出一個體悟分數。**只有帶著真正體悟的卡片才會進入推薦候選池**；純流水帳不會被推薦給任何人。定位是用程式碼強制執行的（詳見文末附錄）。

---

## 〇、核心哲學：只獎勵高成本的真誠行為

我們的推薦系統裡，唯一被採用的互動訊號是：「這位讀者曾經為哪些作者**寫過一張共振卡**」。讚數、停留時間、點擊率——全部被明確拒絕，寫進了系統的 Not-Doing List。

> **為什麼？**
>
> 按讚是零成本的，停留可能只是分心。但為別人的故事**寫下自己的回應**，是深思熟慮、有代價的行為——它是我們能取得的、最強的真實共鳴證據。廉價的訊號一旦被收集，遲早會被優化；被優化的廉價訊號，就是每一個演算法社群走向表演化的起點。

接下來的每一條準則，都是這句話在不同功能上的展開。遇到新的功能爭論時，先問：*這個設計獎勵的行為，是有成本的真誠，還是無成本的表演？*

---

## 一、公開的回應只有一種——成為作者

「要不要做留言區」的爭論，癥結其實不在留言區本身。FB / Threads 式的留言區之所以產生創作者與觀眾的主從關係，是因為它把兩種本質不同的行為混進了同一個公開空間：**對作者說話**（我很感動、我也經歷過），和**對觀眾表演**（搶樓、抖機靈、站隊）。留言區有觀眾，說話就會變成表演。

所以我們不做二選一，而是*按「說話的對象」把回應拆成三層*：

### 【三層回應結構】

1.  **收藏（對自己說）**
    *   **描述：** 純私人、不顯示計數、不通知任何人。
    *   **成本：** 最低
2.  **私下回應（對作者說）**
    *   **描述：** 一張只有作者看得到的小紙條，沒有觀眾、不公開、不計數。
    *   **成本：** 低
3.  **共振（對世界說）**
    *   **描述：** 唯一的公開回應：寫一張自己的卡片。回應者即作者。
    *   **成本：** 高・唯一公開

> *設計哲學：三層回應：對自己說、對作者說、對世界說。公開層只有一種形式。*

這個結構同時回答了三個原本各自為政的問題：

*   **「我很喜歡，已收藏」的出口** —— 收藏與私下回應承接了所有輕量回饋，讀者不再被迫在「寫一張卡」和「沉默離開」之間二選一。
*   **主從關係** —— 讀者對作者的話走私下通道，沒有觀眾就沒有表演，也就沒有台上台下。
*   **作者的安全** —— 寫下脆弱真實故事的人，不必面對公開留言區裡的嘲諷、獵奇與審判。私下回應反而比公開留言更真誠。

### 共振的門檻，用編輯器解，不用留言區解

共振門檻高是事實，但解法不是增設低門檻的公開通道，而是**降低寫作本身的門檻**。點下共振的那一刻，系統早就擁有那張卡的核心體悟——可以讓 AI 用它起頭：「這張卡片的體悟是 X，你有過類似、或完全相反的經驗嗎？」並且允許共振卡很短——三五句話的「回聲」也是一張卡。門檻的本質不是「要寫字」，而是「面對空白編輯器不知從何說起」，而我們手上剛好有解決它的素材。

*額外的機會：私下回應可以成為共振的前奏。作者收到紙條後回一句「這段經驗我很想聽，願意寫成卡片嗎？」——被邀請而寫的第一張卡，比自己鼓起勇氣發的容易得多。*

---

## 二、身份彈性下放到每一張卡

把兩個定位維度放在一起，會得到一個必然的推論：**真實故事常常是脆弱的內容**。一個平常願意用真名的人，寫到離婚、憂鬱症、家庭創傷時可能就不願意了。如果身份彈性只存在於帳號層級，這些體悟濃度最高的故事就不會被寫出來——或者使用者會去開小號，我們反而失去「背後有真實身份、糾紛可調解」的底層保證。

### 【身份三層結構】

*   **▲ 卡片層：可匿名**
    *   單張卡片可選「以匿名發布」・不掛名、不進公開個人頁。
*   **■ 展示層：自選**
    *   暱稱或真名・頭貼放不放・像 Facebook 一樣給選擇。
*   **▼ 帳號層：真實**
    *   註冊時完成真實身份環節・平台可追溯・糾紛可調解。

> *設計哲學：越往下越真實、越往上越自由：信任建立在底層，勇氣發生在頂層。*

*   **帳號層真實** —— 註冊時有真實身份認證環節。這是平台調解糾紛、維持信任的地基，永遠不對外展示。
*   **展示層自選** —— 要用暱稱還是真名、要不要放頭貼，交給使用者。我們傾向真實，但不強迫。
*   **卡片層可匿名** —— 單張卡片可以選擇匿名發布：不掛名、不出現在公開個人頁，但平台後端知道作者是誰。最脆弱的故事，往往就是最值得被寫出來的故事。

*工程注意事項：匿名卡的向量照樣進作者自己的推薦畫像（私人內容本來就只影響本人的推薦），但公開候選池的記錄帶有作者 ID，「共振作者加分」等 join 路徑必須確保不會間接洩露匿名卡的作者身份。*

## 三、關係從故事裡長出來

「遇到心靈契合的人」是我們的目標之一，但**無門檻的通用私訊是重災區**——騷擾、搭訕、詐騙都從那裡開始，而我們的使用者剛剛在平台上袒露了人生最脆弱的部分。所以私訊不是一個平行蓋起來的聊天室，而是*要用故事賺來的東西*：

### 【從故事到私訊的路徑】

```
[ 讀到打中你的故事 ] ───> [ 私下回應或寫共振卡 ] ───> [ 連結邀請，雙方同意 ] ───> [ 私訊開啟 ]
```

> *設計哲學：陌生人之間唯一的接觸路徑——每一段對話的開場白，都是一個具體的故事。私訊藏在關係的最深處，而關係的入口永遠是作品。*

這條路徑剛好搭在我們已有的一對一連結（invite / connection）機制上，不需要新的信任模型。它保證私訊的第一句話永遠有脈絡——不是「嗨，在嗎」，而是「你寫的那張卡」。

### 群組：先不做。想要「一群人」，用主題圈

群聊會把平台的重心從「寫」拉到「聊」，管理成本極高，而且即時訊息天然獎勵快與淺。如果之後想要群體感，更貼合定位的形態是**主題圈／故事集**：一個題目（「第一次送別」「離開穩定工作的那天」），大家各自投入卡片、圈內互相共振。它是非同步的、以作品為中心的聚集——不是聊天室，是一場不散的展覽。這同時也解決了冷啟動時期的內容組織問題。

---

## 四、冷啟動即宣言：先寫，才有讀

我們的推薦畫像是從「你寫過什麼」建立的，不是「你看過什麼」——一個從未寫過卡片的使用者，沒有個人化的 feed。**別把這當成缺陷去修，把它做成入門儀式。**

> **產品含義**
>
> 註冊後的第一件事不是滑 feed，而是用兩三個引導問題，陪新使用者寫出第一張卡。「人人都是創作者」在這裡不是標語，是架構事实：你寫下的第一個體悟，就是這個世界開始為你排列的起點。

---

## 五、AI 是鏡子，不是守門員

系統為每張卡萃取體悟簽名、為每次推薦產生理由。這些能力該怎麼露出，有兩條紅線一條機會：

*   **把推薦理由亮出來。** 選卡的模型本來就會寫下推薦理由——直接顯示在 feed 上：「推薦給你，因為你寫過的某個體悟和這篇呼應」。透明的推薦是對黑盒演算法最鮮明的差異化，也讓使用者理解「我寫什麼，決定我看到什麼」，反過來激勵寫作。
*   **把簽名回顯給作者。** 發布前告訴作者「我們讀到的核心體悟是：……」。寫作者常常自己都沒意識到寫出了什麼——這是一面鏡子，也是很動人的產品時刻。
*   **但永遠不顯示體悟分數、不說「體悟不足」。** 分數只在後台安靜地決定一張卡要不要進推薦池。AI 沒有資格公開裁判一個人的人生領悟。

---

## 六、指標必須跟上價值觀

程式碼層面我們已經拒絕了讚數與停留時間，但如果營運指標還在看 DAU 和 time-spent，遲早會有壓力把讚數加回來。北極星指標應該量我們真正想要的東西：

### 【指標對照】

| 我們不量 (❌) | 我們量 (✅) |
| :--- | :--- |
| DAU・停留時間 | 每位讀者每週寫出的共振卡數 |
| 讚數・點擊率 | 私下回應 → 共振的轉化率 |

> *設計哲學：量表演，得到表演；量真誠，才有機會得到真誠。*

## 附錄：體悟推薦系統現況

供還沒看過程式碼的成員參考——上述準則不是憑空的哲學，是從已經運作的架構裡長出來的。系統分成寫入端與讀取端，成本刻意壓在寫入端（每張卡一生只做一次）：

### 【推薦系統管線圖】

```
【寫入端（每張卡一次）】

  [ 卡片發布 ] 
       │
       ▼
  [ AI 萃取體悟簽名 ] ── (核心體悟、處境、情緒、分數)
       │
       ▼
  < 體悟門檻 > ─── (通過) ───> [ 向量庫 (候選池) ]
       │
       └── (未通過) ───> 不推薦給任何人
```

```
【讀取端（每次打開 feed）】

  [ 向量庫 (候選池) ]
       │
       ▼  (1) 向量檢索：從你的體悟出發，撈 200 個候選
  [ 200 個候選 ]
       │
       ▼  (2) 輕量模型重排，篩選保留 30 個
  [ 30 個候選 ]
       │
       ▼  (3) 強模型精選 8 個 ＋ 附上推薦理由
  [ 最終推薦 8 個 feed 項目 ]
```

> *設計哲學：寫入端：發布 → 萃取簽名 → 過門檻才入池。讀取端：三層漏斗，越窄越貴。*

幾個支撐前文準則的既有設計：

*   **使用者畫像來自書寫。** 把使用者自己所有卡片（含私人卡）的體悟向量聚成幾個中心點——一個人是一組體悟方向，不是一個平均值。私人卡只影響本人的推薦，永不外洩。
*   **唯一的互動加分是共振。** 你曾為哪些作者寫過共振卡，他們的新作會得到輕微加分。讚數、停留時間在 Not-Doing List 上。
*   **內建反同溫層旋鈕。** 系統預留了「相同處境、不同體悟」的檢索通道（v1 暫時關閉）——未來可以刻意引入有生產性的對照，而不是只餵相似的聲音。

---

> 公開互動只有成為作者一途；
> 輕的回饋轉為私下、無計數；
> 身份的彈性下放到每一張卡；
> 關係，必須經由故事建立。
